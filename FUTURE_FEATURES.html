A. חובה ל-MVP אמיתי — אסור לעלות לאוויר בלי זה

A1. Onboarding + KYC ★★★

זמן: 3-4 שבועות מצב נוכחי: אין — האפליקציה נכנסת ישר ל-Home עם משתמשת מדומה צריך: מסכי הרשמה (טלפון → SMS OTP → סלפי + ת.ז. דרך Persona/Onfido/Veriff → אישור עוסק מורשה ברשות המסים → תעודה דיגיטלית) למה לבנק: ללא KYC מלא הבנק לא יכול לפתוח חשבונות. זה התנאי הראשון לרישוי PSP/EMI.

A2. אינטגרציה אמיתית מול API "חשבוניות ישראל" ★★★

זמן: 6-8 שבועות (כולל תהליך רישום מול רשות המסים) מצב נוכחי: mock — אנימציה של 3 שניות, מספר רנדומלי צריך: חיבור אמיתי ל-SHAAM API, ניהול תעודה דיגיטלית, retry logic, error handling, logging לכל הקצאה למה לבנק: זה ה-USP. ללא אינטגרציה אמיתית — הסיפור שלך מתפרק

A3. חיבור לחשבון בנק ישראלי (PFM / Open Banking) ★★★

זמן: 4-6 שבועות מצב נוכחי: אין — היתרות מומצאות צריך: Open Banking API (לפי תקנת PSD2 הישראלית, שעוד תיכנס לתוקף בהדרגה), או חיבור ישיר לחשבונות הפועלים/דיסקונט/לאומי לפי הבנק התומך למה לבנק: בלי חיבור לחשבון בנק, המשתמש לא יכול להוציא/להפקיד כסף אמיתי

A4. תשלום אמיתי מקבלת USD/EUR מחו"ל ★★★

זמן: 4-8 שבועות (תלוי בשותף) מצב נוכחי: אין — רק QR mock צריך: או Stripe Connect (קל יותר, עמלה גבוהה ~3%), או חשבון IBAN וירטואלי דרך Wise Platform / Currencycloud / Banking-as-a-Service partner למה לבנק: זה הסעיף שמייצר את ההכנסה לבנק מ-FX margins

A5. אבטחה: 2FA + Device Management ★★★

זמן: 2-3 שבועות מצב נוכחי: ה-toggle של ביומטרי שם, אבל לא עושה כלום צריך: TOTP (Google Authenticator), SMS fallback, רשימת מכשירים מורשים עם אפשרות revocation למה לבנק: דרישה רגולטורית של בנק ישראל — בלי 2FA אמיתי הבנק יסרב

A6. AML / Fraud Detection מובנה ★★★

זמן: 4-6 שבועות (כולל אינטגרציה) מצב נוכחי: אין צריך: Chainalysis / Elliptic / ComplyAdvantage — סינון בכל עסקה בינלאומית, גיוון לפי מדינות OFAC, alerts ל-compliance officer למה לבנק: תנאי רישוי. בלי AML המוצר לא יקבל אישור

B. חזק מאוד ל-MVP — ייצר תלות גבוהה ושימוש יומיומי

B1. ניהול חשבונות יורו/דולר נפרדים בארנק ★★★

זמן: 2 שבועות (אם A4 קיים) מצב נוכחי: היתרות מוצגות אבל לא אינטראקטיביות צריך: לחיצה על יתרת USD → פתיחת עמוד dedicated עם histroy + תפעול ספציפי לאותו מטבע למה לבנק: ערך אמיתי לעצמאי, מבדל ברור מ-Bit

B2. גישת רואה חשבון (Accountant Portal) ★★★

זמן: 3-4 שבועות מצב נוכחי: אין צריך: הזמנת רו"ח באימייל, מתן הרשאת קריאה בלבד, ייצוא חשבוניות/הוצאות לחשבשבת/רווחית/EZcount למה לבנק: כל עצמאי ישראלי עובד עם רו"ח — אם לרו"ח קל לעבוד עם GlobalBiz, השוק נסגר

B3. תשלומים חוזרים (Recurring) ★★

זמן: 2-3 שבועות מצב נוכחי: אין צריך: הגדרת תשלום חודשי לקבלן משנה (1 לכל חודש), חשבונית ריטיינר חודשית ללקוח, alerts לפני למה לבנק: ARPU גבוה — עצמאים עם ריטיינרים = משתמשים יציבים

B4. דוח הוצאות + קיזוז אוטומטי ★★★

זמן: 3 שבועות מצב נוכחי: AI insight stub צריך: זיהוי אוטומטי של הוצאות עסקיות (Figma, Vercel, Adobe, AWS), שיוך לקטגוריה, חישוב קיזוז מע"מ + מס הכנסה, ייצוא דוח חודשי למה לבנק: חוסך לעצמאי שעות עבודה בחודש — value prop חזק לאחיזה

B5. חשבוניות מרובות (Bulk Invoicing) ★★

זמן: 2-3 שבועות מצב נוכחי: רק חשבונית בודדת צריך: טעינת CSV של לקוחות + סכומים, יצירת N חשבוניות בבת אחת, N הקצאות במקביל מרשות המסים למה לבנק: סוכנויות/סטודיואים עם 20+ לקוחות חודשיים יעברו אלינו רק בזכות זה

B6. כרטיס Debit Business ★★★

זמן: 6-10 שבועות (תלוי ב-BIN sponsor) מצב נוכחי: אין צריך: הנפקת כרטיס וירטואלי + פלסטיק דרך BIN sponsor (Bank Hapoalim International? Discount IPS?), אפשרות freeze/unfreeze, מגבלות יומיות, התראות למה לבנק: מקור הכנסה ישיר (interchange ~1.5%) + תלות עמוקה

B7. שותפויות לקבלני משנה ("Sub-contractor groups") ★★

זמן: 3-4 שבועות מצב נוכחי: אין — תשלום בודד בלבד צריך: יצירת קבוצה (פרויקט), חלוקת תשלום בין N קבלני משנה, גם תיעוד טופס 856 לרשות המסים למה לבנק: ייחודי — אף אחד לא עושה את זה. סטודיואים יחפשו את זה.

C. נחמד שיש — שיפור חווית משתמש

C1. סריקת קבלות (OCR) ★★

זמן: 2-3 שבועות צריך: מצלמה → Tesseract / Google Vision API → אוטו-מילוי שדה הוצאה למה: מהיר יותר מהזנה ידנית

C2. צ'אט תמיכה מובנה ★

זמן: 1 שבוע (אם משתמשים ב-Intercom/Crisp) צריך: widget בתוך האפליקציה למה: הפחתת churn

C3. מסכי דשבורד גרפיים ★

זמן: 2-3 שבועות צריך: Victory Native / React Native SVG charts — גרפים של הכנסות, הוצאות, מגמות למה: אסתטיקה לדמו, עומק לעו"ש העצמאי

C4. התראות Push אמיתיות ★★

זמן: 1-2 שבועות מצב נוכחי: רק toggle ב-Profile צריך: Expo Notifications + backend trigger בקבלת תשלום / איחור חשבונית / ריבית הצטברה למה: retention, engagement

C5. מצב כהה/בהיר ★

זמן: 1 שבוע מצב נוכחי: רק כהה צריך: toggle בהגדרות + רספונסיביות לפלטה החדשה למה: העדפה אישית

D. חדשנות פוטנציאלית — פיצ'רים שאף אחד לא עושה

D1. AI Assistant בטקסט חופשי ★★★

זמן: 4-6 שבועות רעיון: "שלם לאורי 1800 על backend" → האפליקציה מבינה, פותחת flow מוכן עם השם, הסכום, התיאור — רק ללחוץ confirm למה זה ענק: Bit/Paybox צרצרים שזקוקים ל-5 לחיצות. כאן זה משפט אחד. איך: Claude API → JSON structured output → router.push למסך הזנת סכום ממולא

D2. Cashback אוטומטי על הוצאות עסקיות ★★

זמן: 4 שבועות (דורש partnerships) רעיון: 1.5% cashback על תשלום ל-Figma, AWS, Vercel, וכד' (BIN partners) למה: הכנסה לבנק + סטיקיניות

D3. Invoice Factoring / Credit Line ★★★

זמן: 8-12 שבועות רעיון: חשבונית פתוחה של ₪15K → קבלת ₪13K עכשיו, עם ריבית 1.5% חודשי. הבנק מקבל את הכסף מהלקוח ישירות. למה לבנק: מוצר מימון רווחי + פתרון cash flow לעצמאי זה ענק — זה הופך את GlobalBiz מאפליקציית תשלומים לגוף מימון מיני לעצמאים

D4. Currency Hedging — נעילת שער ★

זמן: 4 שבועות רעיון: לקוח אמריקאי משלם בעוד 30 יום $5,000 — אפשר לנעול שער עכשיו תמורת עמלה קטנה (0.3%) למה: הגנה מתנודתיות + עמלה לבנק

D5. שוק עצמאים פנימי (Marketplace) ★

זמן: 6-8 שבועות רעיון: עצמאים על GlobalBiz מציעים שירותים זה לזה (network effect) למה: Lock-in גבוה, גם הפניות בתוך המערכת

D6. סוכן מס AI — דיווח אוטומטי לרשות המסים ★★★

זמן: 6-10 שבועות רעיון: דוח 6111 / טופס 1301 מוכן אוטומטית בסוף שנה, עם המלצות אופטימיזציה למה: ערך כספי ברור (חוסך לעצמאי ₪3,000-8,000 בשנה בשירותי רו"ח)

הערכת השקעה כוללת

Tier זמן פיתוח עלות פיתוח תוצאה
Tier 1 (יסודות) 3 חודשים ₪400-600K אפליקציה לגיטימית, רישוי PSP
Tier 2 (כסף אמיתי) 3 חודשים ₪500-800K מוצר רב-מטבעי שעצמאי משתמש בו
Tier 3 (גידול) 4 חודשים ₪700K-1M מוצר עם מבדלים אמיתיים
Tier 4 (מהפך) 6 חודשים ₪1-2M פלטפורמת fintech מלאה

סה"כ ל-MVP חזק: 16 חודשים, ₪2.6-4.4M

לבנק שמשקיע — ROI: כפי שמופיע במסמך הניתוח התחרותי, ARPU של ₪1,518/שנה × 30,000 משתמשים פעילים = ₪45.5M/שנה. החזר ההשקעה תוך 18-24 חודשים מהשקת ה-Tier 2.

מה אני הכי ממליץ להוסיף עכשיו לדמו

אם אתה רוצה לחזק את הדמו לבנק בלי לבנות backend (להישאר ב-prototype):

  1. מסך KYC mock — onboarding flow מצולם בלבד שמראה: סלפי → צילום ת.ז. → אישור עוסק → תעודה דיגיטלית (5 מסכים, ~3 ימים)
  2. AI Assistant in-app — שורת חיפוש "?כיצד אוכל לעזור" שמובילה ל-flow מהיר (mock עם הכנות). אפשר אפילו ב-Claude API אמיתי (~2 ימים)
  3. דשבורד "פרוייקטים פעילים" — רשימת 3-4 פרוייקטים עם שעות, חשבוניות, רווחיות (~2 ימים)
  4. מסך Invoice Factoring — "החשבונית של Berlin Studio (€3,800) — קבל ₪14,500 עכשיו במקום לחכות 30 יום. ריבית: 1.2%" (~1 יום)

ה-4 פיצ'רים הללו לוקחים שבוע עבודה ונותנים לדמו רוחב פיצ'רים שאף אחד מהמתחרים לא מתקרב אליו.
